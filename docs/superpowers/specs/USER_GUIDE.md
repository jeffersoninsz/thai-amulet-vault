## 🚀 一、启动与运行

### 日常启动
在项目根目录找到 **`start-dev.bat`**，**双击即可**。该脚本集成了智能网络嗅探，会自动获取物理 IP 并配置环境变量，确保局域网（包括手机端）可直接访问。

启动成功后，浏览器访问：
- **前台商城**: `http://localhost:3000`
- **后台管理**: `http://localhost:3000/admin`

### 手动启动 (备选)
打开终端，进入项目目录，执行：
```
npm run dev
```

> **常见问题**：如果启动报错"端口 3000 已被占用"，先关闭之前的终端窗口再双击 `start-dev.bat`，脚本会自动处理。

## 👨‍💼 三、后台管理操作指南

### 3.1 圣物管理 (核心功能)

**入口**: 登录后台 → 左侧导航「资料库后台」

#### 新增圣物
1. 点击右上角 **「➕ 新增佛牌档案」**
2. 填写必要字段：中文名、英文名、高僧/寺庙、年份、材质
3. 点击 **「Select File to Upload」** 上传图片（会弹出 Cloudinary 上传弹窗）
4. 设置价格、库存、起订量
5. 如需设为批发专属，勾选 **「B2B Only」**
6. 点击提交，页面会弹出绿色 ✅ 提示

#### 编辑现有圣物
1. 在列表中找到目标圣物，点击 **「编辑档案」**
2. 修改任意字段（名称、描述、价格等）
3. 如需更换图片，点击 **「Select File to Upload」** 重新上传
4. 点击保存图标 💾，成功后会弹出 **「档案更新成功！」** 提示

#### 删除圣物
1. 点击 **「删除」** 按钮
2. 系统会弹出二次确认对话框
3. 确认后该记录将被永久删除（**无法撤销**）

### 3.2 图片上传说明

系统使用 **Cloudinary** 云端图床，上传后图片自动托管在全球 CDN。

**上传步骤**：
1. 在编辑表单中，点击 **「Select File to Upload」**
2. 从弹出的 Cloudinary 窗口中选择本地图片文件
3. 等待上传完成（按钮会显示 Uploading... 状态）
4. 上传成功后，左侧预览区会即时显示新图
5. **必须点击保存**才会将新图片 URL 写入数据库

> **图片规格建议**：
> - 产品图：800×800px 以上，1:1 正方形，纯黑背景 (#0d0c0b)
> - 文件大小：10MB 以内
> - 格式：JPG / PNG（系统自动转 WebP 优化）

### 3.3 文章/资讯管理

**入口**: 后台 → 「资讯中心」

- **新增文章**: 填写中英文标题、正文内容、分类、封面图
- **发布控制**: 通过「已发布」开关控制文章是否前台可见
- 文章封面图同样通过 Cloudinary 上传

### 3.4 用户管理

**入口**: 后台 → 「用户管理」（仅 ADMIN/SUPER_ADMIN 可见）

- 查看所有注册用户列表。
- **修改角色**: 将普通用户提升为 WHOLESALE 批发商，或设为 STAFF 员工。
- **注销档案**: 点击红色垃圾桶图标，确认后可永久删除用户帐号。
- **安全性**: 
    - 严禁删除当前登录的超级管理员帐号（系统已自动禁止）。
    - 所有的注销操作都会在「系统日志」中留下永久审计记录。

### 3.5 系统设置

**入口**: 后台 → 「系统设置」

可配置内容包括：
- 在线人数显示开关与基准值
- 营销弹窗（假销量通知）的城市池与间隔频率
- 全局公告与页脚文案

## 🎨 四、设计师协作指引

设计师只需提供高质量素材，**无需接触任何代码**。

### 素材规格表

| 资产类型 | 推荐尺寸 | 比例 | 背景建议 |
| :--- | :--- | :--- | :--- |
| PC 首页 Banner | 1920×600 px | 3.2:1 | 深色渐变融入网站 |
| 手机版 Banner | 1080×1440 px | 3:4 | 主体靠上展示 |

### 协作流程
1. **设计**: 按上述规格出图，导出 JPG/PNG
2. **交付**: 将图片发送给管理员，由管理员在后台上传
3. **微调**: 如需裁切调整，可登录 [Cloudinary 控制台](https://console.cloudinary.com) 直接编辑，网站会自动刷新

## 🌐 五、部署方案

### 本项目技术栈
- **框架**: Next.js 16 (App Router + Server Actions)
- **数据库**: SQLite (通过 Prisma ORM)
- **图床**: Cloudinary CDN
- **认证**: NextAuth.js (JWT)

### 部署到 Hostinger

> **注意**: Hostinger 的普通虚拟主机 (Shared Hosting) **无法运行**本项目。

| 方案 | 适用性 | 说明 |
| :--- | :--- | :--- |
| **VPS 主机** (推荐) | ✅ 完美支持 | 需安装 Node.js，用 PM2 管理进程 |
| **Node.js Hosting** | ✅ 支持 | Hostinger 新推出的托管服务，支持 GitHub 一键部署 |
| **共享主机** | ❌ 不支持 | 仅支持 PHP，无法运行 Node.js |

### 部署到 Vercel (最简方案)
```
npm i -g vercel
vercel --prod
```
Vercel 是 Next.js 的原厂部署平台，免费额度充足，部署最简单。

## 🚨 六、常见问题与故障排查

### Q: 图片显示不出来（裂图）？
**A**: 检查终端是否有 `resolved to private ip` 的报错。如果有，说明你的 VPN/代理干扰了 Cloudinary 的域名解析。当前已通过 `next.config.ts` 中的 `unoptimized: true` 修复此问题。关掉 VPN 或重启服务后刷新页面即可。

### Q: 上传图片后保存没反应？
**A**: 请检查浏览器控制台 (F12) 是否有错误。成功保存应当弹出绿色提示「档案更新成功！」。如未弹出，检查终端日志中是否有 `[updateAmuletAction] FAILED` 的输出。

### Q: 登录失败？
**A**: 确保已执行 `npx tsx scripts/seedAccounts.ts` 创建测试账户。默认密码为 `password123`。

### Q: 批量图片失效？
**A**: 运行灾难恢复脚本：
```
node scripts/reupload-from-json.js
```
